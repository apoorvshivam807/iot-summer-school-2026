## Q10. (Theory) What is a `.gitignore` file? 

Write a `.gitignore` for an Arduino project that excludes compiled output files (`.hex`, `.elf`), OS-specific files (`.DS_Store`, `Thumbs.db`), and IDE config folders (`.vscode/`, `build/`). 

### 1. Technical Explanation

A `.gitignore` file is a structural plain text file saved directly at the root level of your Git project workspace. It informs the Git tracking daemon which specific file matching patterns, compiled extensions, or local build directories it should completely ignore.

Using a `.gitignore` file is critical to ensure that heavy compiled binaries, temporary operating system cache logs, and local IDE environment layouts do not bloat your history or accidentally leak private tracking parameters into your public public repository.

### 2. Target Arduino `.gitignore` Implementation

Create a file named exactly `.gitignore` and add the following lines to satisfy your assignment constraints:

```text
# ====================================================================
# IIT Jammu Summer School 2026 - Arduino/C++ Project Exclusions
# ====================================================================

# 1. Compiled Binary Output Files
*.hex
*.elf

# 2. Operating System Specific Files
.DS_Store
Thumbs.db

# 3. IDE Configuration and Build Directories
.vscode/
build/

```
