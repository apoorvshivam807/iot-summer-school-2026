## Q11. IoT Architecture

An IoT system is structured into a hierarchical multi-layer framework that facilitates data collection, transmission, processing, and application.

```
+-------------------------------------------------------------+
|                      APPLICATION LAYER                      |
|   (Smart Agriculture Dashboards, Industrial Monitoring UI)   |
+-------------------------------------------------------------+
                              ▲
                              │
+-------------------------------------------------------------+
|                      PROCESSING LAYER                       |
|         (Cloud Servers, Database Engines, Analytics)        |
+-------------------------------------------------------------+
                              ▲
                              │
+-------------------------------------------------------------+
|                        NETWORK LAYER                        |
|             (Wi-Fi, MQTT Brokers, Cellular, LoRa)           |
+-------------------------------------------------------------+
                              ▲
                              │
+-------------------------------------------------------------+
|                       PERCEPTION LAYER                      |
|         (Sensors, Actuators, Hardware Microcontrollers)     |
+-------------------------------------------------------------+

```

Explanation of the 4 Layers:

1. 
**Perception Layer (Sensors/Actuators):** This physical layer interacts directly with the environment to capture analog or digital data or perform actions. Example technology: DHT11 Temperature & Humidity Sensor.


2. 
**Network Layer (Data Transmission):** Responsible for transmitting the collected sensor data from the hardware node to the processing center safely and securely. Example technology: Wi-Fi (ESP32/ESP8266) or MQTT protocol.


3. 
**Processing Layer (Data Analysis):** Consolidates, processes, stores, and analyzes the incoming mass data streams. Example technology: HiveMQ Cloud Broker / Database Systems.


4. 
**Application Layer (User Interface):** The front-end interface where data is visually presented to users and from which operational commands are issued. Example technology: Adafruit IO Smart Dashboard App.
