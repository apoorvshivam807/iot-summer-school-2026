# Q29. (Theory) Sensor Calibration & Two-Point Soil Moisture Calibration Explained

This document details the definition and importance of sensor calibration within Internet of Things (IoT) ecosystems, alongside a step-by-step methodology for executing a two-point calibration on an analog soil moisture sensor to map its raw readings into accurate percentage metrics.

---

## 1. What is Sensor Calibration and Why is it Important?

### Definition
**Sensor Calibration** is the operational process of comparing an uncalibrated, raw sensor output signal against an established, known physical standard or absolute baseline reference. This process determines the relationship between the raw electrical parameter measured by the system (such as voltage, resistance, or digital counts) and the actual physical unit of interest (such as temperature, humidity, or volumetric water content).

### Importance in IoT Systems
* **Correcting Manufacturing Tolerances:** Mass-produced sensors exhibit variance from the factory; minor physical or chemical deviations mean two identical sensor probes can output different raw values under identical conditions.
* **Compensating for Degradation and Drift:** Over time, physical sensor components undergo mechanical wear, chemical oxidation, or environmental fouling, which alters their baseline electrical properties.
* **Ensuring Interoperability:** Standardizing raw data across an entire IoT fleet guarantees that an absolute threshold rule (e.g., "trigger irrigation when moisture drops below 30%") executes identically across all deployed hardware nodes.
* **Converting to Engineering Units:** Raw ADC integer steps ($0$–$1023$) carry no physical meaning to end-users or data analytics software until they are mapped to real-world units (like $\%$, $^\circ\text{C}$, or $\text{cm}$).

---

## 2. Two-Point Calibration Method for an Analog Soil Moisture Sensor

An analog soil moisture probe operates by measuring the dielectric permittivity of the surrounding medium, outputting a variable analog voltage that the Arduino’s 10-bit ADC digitizes into a raw integer range (typically spanning between an upper dry limit and a lower saturated wet limit). 

To convert these raw integer values into a clear, accurate **$0\%$ to $100\%$ percentage scale**, we apply the **Two-Point Calibration Method**.



### Step-by-Step Calibration Procedure:

1. **Establish Point 1 (The Completely Dry Baseline - 0%):**
   * Wipe the soil moisture probe completely dry and hold it suspended in open air.
   * Read the Serial Monitor output and record the raw ADC value. Because air provides minimum conductivity, this condition yields the highest resistance and largest raw value (for example: $1023$).
   * Designate this value as your baseline for **$0\%$ Moisture** ($\text{Raw}_{\text{dry}}$).

2. **Establish Point 2 (The Completely Saturated Wet Baseline - 100%):**
   * Submerge the sensor probe entirely into a glass of clean water up to its maximum safe depth line.
   * Monitor the Serial Monitor output and record the lowest stable raw ADC integer step. Water acts as a highly conductive path relative to air, dropping sensor resistance and bringing the voltage reading close to its minimum (for example: $250$).
   * Designate this value as your baseline for **$100\%$ Moisture** ($\text{Raw}_{\text{wet}}$).

3. **Software Mapping and Implementation:**
   * Once these two extreme reference boundaries are locked in, apply a linear mapping function within your Arduino `loop()` code block. The built-in `map()` function re-scales the live sensor input between the dry reference and wet reference down to a percentage range:

```cpp
// Explicit hardware calibration coefficients discovered during testing
const int DRY_REFERENCE = 1023;  // Value in pure open air (0% moisture)
const int WET_REFERENCE = 250;   // Value submerged in water (100% moisture)

void setup() {
  Serial.begin(9600);
  pinMode(A0, INPUT);
}

void loop() {
  // Sample the raw uncalibrated ADC potential from the analog channel
  int rawReading = analogRead(A0);

  // Apply the two-point linear mapping logic
  // Note: We reverse the boundaries because lower raw values mean wetter soil conditions
  int moisturePercentage = map(rawReading, DRY_REFERENCE, WET_REFERENCE, 0, 100);

  // Constrain the output values between 0 and 100 to filter outlier noise spikes
  moisturePercentage = constrain(moisturePercentage, 0, 100);

  // Print the calibrated telemetry out to the serial interface
  Serial.print("Raw Signal: ");
  Serial.print(rawReading);
  Serial.print(" | Calibrated Moisture: ");
  Serial.print(moisturePercentage);
  Serial.println("%");

  delay(1000);
}
