## Q9. (Theory) Explain the difference between `git pull`, `git fetch`, and `git clone`. 

When would you use each? 

### 1. `git clone`

* 
**Technical Definition:** Downloads a complete copy of an existing remote repository from GitHub down to your local computer. It automatically initializes the local `.git` system folder, establishes remote-tracking configurations, and establishes a default link back to the online source repository hosted on GitHub.


* 
**Primary Use Case:** You execute this command exactly once when beginning work on a project repository for the **very first time** on your laptop or machine configuration.



### 2. `git fetch`

* 
**Technical Definition:** Contacts the remote host repository and downloads all new metadata tracking points, code references, and branch logs that others have pushed. Crucially, it **does not merge** or alter any of your local workspace files or your active branch code. It merely brings the local tracking ledger up to date.


* 
**Primary Use Case:** You execute this when you want to safely review remote project updates and branch changes made by classmates or peers **before integrating them** into your active local files.



### 3. `git pull`

* 
**Technical Definition:** An automated, combined two-step command sequence that immediately fetches updates from the remote host and merges them directly into your active local working branch. Mathematically, it operates as:



$$\text{git pull} = \text{git fetch} + \text{git merge}$$


* 
**Primary Use Case:** You execute this daily to **keep your local repository synchronized** with the main branch on GitHub before starting to write or commit new code modifications.
